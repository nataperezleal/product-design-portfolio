Proyecto 01

Control de estados críticos del sistema

Bizagi Studio · Enterprise UX · Critical Actions · System Feedback

El problema real
Los administradores necesitaban controlar estados del entorno — mantenimiento, reinicio de servicios — pero el sistema no comunicaba qué pasaría ni cuándo. Ejecutar una acción crítica sin feedback claro en producción no es solo un problema de UX: es un riesgo operativo. Adicionalmente, no existía ninguna capa de comunicación hacia usuarios finales afectados.
El proceso
Identifiqué dos problemas que el equipo trataba como uno: el control del sistema y la comunicación con usuarios finales. Separarlos fue la decisión de diseño más importante. Exploré variantes donde ambas capas estaban combinadas — y resultaban confusas — antes de definir un flujo que las separa con claridad sin añadir pasos.
La solución
Sistema de tres estados con feedback visual inequívoco, integrado con un módulo de comunicación que permite notificar a usuarios antes de ejecutar la acción. La comunicación se diseñó como parte del flujo de control, no como una feature separada.
Aprendizaje de diseño
En herramientas enterprise, el feedback del sistema no es secundario: es la feature principal. Diseñar primero el estado y luego la acción —no al revés— fue lo que desbloqueó el flujo correcto.
